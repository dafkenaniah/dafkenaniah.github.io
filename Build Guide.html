Lead's Guide to Build Management & Troubleshooting

A playbook for managing the daily build cadence and resolving common issues, based on the MLB '25 cycle.

The Daily Build Cadence

The project's rhythm is dictated by the daily delivery of new builds. Understanding this flow is critical for planning your team's day.

  • Morning Delivery: Expect new builds to be delivered between 9:00 AM and 12:00 PM PST. The Build Management team (Cristian, Chris, Aleesha, etc.) will post a message in the `mlb-builds` channel for each platform.
  • Standard Naming Convention: Builds follow a clear pattern: `MLB25 (Platform) Build#_Date_CL#`. For example: `MLB25 (PS5) 15_22-OCT-24_CL911664`. This is crucial for bug reporting and tracking.
  • Location: Builds are delivered to two primary locations: the `Current_Builds` network share and `AoC` (Aspera on Cloud). Your team should be familiar with downloading from both.
  • Mid-Day Patches: Be prepared for unexpected mid-day patch deliveries, especially for critical fixes. These are often announced in the main channel and require your team to be flexible and pivot to test them immediately.

Troubleshooting Playbook

Issues with build delivery are common. Here’s how to handle the most frequent problems based on real events from the cycle.

Scenario 1: A Build is Missing

  • Problem: A specific build component is missing (e.g., 'PS5 UP-9000 mainline build', 'NSW Test patch', 'Xbox AUTO patch').
  • Action: First, check the main delivery folder on the network share, as sometimes it's delivered but not announced. If it's not there, post a clear, polite message in the `mlb-builds` channel. Tag the Build Manager on duty (e.g., `@cristian.pena`).
  • Example Request: `Hey @cristian.pena, could we get the XB2 auto patch uploaded when you can?`

Scenario 2: Incorrect Build Version

  • Problem: The build in the folder or on the Xbox store has a different version number than expected (e.g., seeing `.28` when expecting `.29`).
  • Action: This is often a publishing issue. First, confirm the version you are seeing. Then, post in the main `mlbqa-pd-link` channel and tag a producer (e.g., `@marcus.efting`, `@cody.sheppard`). They will coordinate with the release manager ('Jason') to publish the correct version.

Scenario 3: Reverting to a Previous Build

  • Problem: A new build is too unstable to test effectively, and the decision is made to revert.
  • Action: This is a high-priority request. Post in the `mlb-builds` channel with a clear request. Tag the Build Manager and specify the exact build you need restored to both the network share and AoC.
  • Example Request: `Hey @cristian.pena, Could we get the builds from Thursday back onto current builds folders, we're reverting back to those. AoC too please.`
