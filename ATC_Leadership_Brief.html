What It Is

A centralized desktop hub that launches and manages two tools: (1) automated video capture for bug reproduction moments, and (2) AI‑assisted “Steps to Reproduce” (STR) generation with a lightweight quality feedback loop.

  • Bugit Monitor — automates OBS replay recording at the moment of interest; standardizes video evidence.
  • Bug Monitor AI — drafts STRs from bug summaries and supports reviewer scoring to improve future outputs.
  • Launcher GUI — single pane of glass for start/stop, logs, config, and dependency install.

Why It Matters

  • Faster, more consistent bug reports
  • Higher STR quality with lower rework
  • Automation replaces repetitive steps (clip, attach, structure)
  • Built-in metrics and auditability

Deployment Overview

  • Deliverable: Single EXE (PyInstaller) + Scripts folder (Windows 10/11)
  • Install: Open EXE ➝ click “Install Dependencies” ➝ configure JIRA/OBS ➝ start tools
  • Update: Replace EXE + Scripts; configs and logs preserved
  • Rollback: Keep prior deployment folder; swap if needed

Rollout Plan

  • Week 0 Smoke test on 2–3 machines
  • Week 1 Internal pilot — 4 testers; tune settings and gather KPIs
  • Week 2 Vendor pilot — 3 testers; validate access and environment
  • Week 3 Vendor expansion — +4 testers (total 7 vendors); monitor KPIs and stabilize
  • Week 4 Optional vendor expansion — +4 testers (total 11 vendors) based on KPIs
  • Week 5+ Diagnostics, config backup/restore, ROI review

KPIs & Measurement

  • Time to file (median, p90)
  • STR first‑pass approval rate
  • AI vs user score delta; rework rate
  • Video attachment success rate
  • Throughput (bugs/tester/day)

Data sources: app logs, AI generation logs, user scoring CSV, optional JIRA fields.

Targets (first quarter)

  • 50–60% median time reduction
  • ≥70% STR approved on first pass
  • ≥90% auto‑clip availability
  • ≤20% rework rate by week 6

Key Risks & Mitigations

Dependency friction on end machines — Use Microsoft Store Python and the in‑app installer (no admin required).

Video capture reliability — Validate OBS replay setup; ensure recording directory/space; provide quick health checks.

JIRA/API auth issues — Clear error messaging and config checks; tokens kept out of logs.

Model/library weight — CPU‑only acceptable for STR text; document install times; cache where possible.

Appendix — What’s Included

  • Bug Monitor AI — STR generation, user scoring CSV, generation logs; YAML config
